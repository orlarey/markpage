# Task lists

GitHub-style task lists: `- [ ]` for an open item, `- [x]` for a done
one. They render as checkboxes.

- [x] Write the spec
- [x] Implement the parser
- [ ] Add the tests
- [ ] Ship it

Nested under a regular list:

1. Planning
   - [x] Scope agreed
   - [ ] Budget signed off
2. Build
   - [ ] Backend
   - [ ] Frontend
